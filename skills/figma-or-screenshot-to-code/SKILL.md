# Figma / Screenshot to Code Skill

## 触发场景

- Figma 设计稿还原
- 截图还原前端页面
- 参考网站 UI 复刻
- 设计稿到代码转换

## 执行流程

1. **分析视觉层级**
   - 识别背景层、容器层、内容层
   - 区分卡片、按钮、文字、图标、间距
   - 标注颜色、字体、阴影、圆角

2. **确认数据边界**
   - 不随意改文案
   - 不随意改数据字段
   - 标注哪些是静态内容、哪些是动态数据

3. **静态还原**
   - 先用硬编码数据还原页面
   - 使用项目已有的 UI 库和组件
   - 确保视觉一致性
   - 优先移动端适配

4. **接入数据**
   - 替换硬编码为实际 API 数据
   - 添加 loading / empty / error 状态
   - 确保数据流正确

5. **截图对比**
   - 对比还原结果与设计稿
   - 记录差异点
   - 逐项修正

## 还原规范

- 间距使用 4px / 8px 倍数
- 字体大小遵循设计稿
- 颜色使用 CSS 变量或 Tailwind 类名
- 图片使用占位图或实际资源
- 交互效果（hover / active / focus）必须实现

## 禁止行为

- 不要跳过视觉分析直接写代码
- 不要随意改变设计稿的布局
- 不要忽略移动端适配
- 不要在静态还原阶段接入真实数据
