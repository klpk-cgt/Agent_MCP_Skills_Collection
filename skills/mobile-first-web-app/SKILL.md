# Mobile First Web App Skill

## 触发场景

- 移动端 H5 开发
- 微信内嵌网页
- 移动端优先的 Web 应用
- 响应式网站开发

## 设计规范

1. **断点策略**
   - 默认：375px / 390px（iPhone 标准）
   - 中屏：768px（iPad 竖屏）
   - 大屏：1024px+（桌面端，仅做增强布局）

2. **触摸交互**
   - 点击区域不小于 44px × 44px
   - 按钮间距不小于 8px
   - 避免悬停依赖的交互

3. **布局原则**
   - 信息卡片化
   - 适配底部 Tab Bar
   - 避免复杂表格，改用卡片列表
   - 使用 sticky 定位固定头部/底部

4. **字体与间距**
   - 正文 14-16px
   - 标题 18-24px
   - 卡片间距 12-16px
   - 内边距 16px

## 执行流程

1. **确认设计稿**
   - 获取移动端设计稿
   - 确认底部 Tab 结构
   - 确认页面跳转方式

2. **搭建布局**
   - 使用 Flexbox / Grid 布局
   - 设置 `viewport` meta 标签
   - 配置安全区域（`safe-area-inset`）

3. **组件开发**
   - 优先卡片式组件
   - 实现下拉刷新
   - 实现上拉加载
   - 添加骨架屏

4. **大屏适配**
   - 使用媒体查询增强布局
   - 大屏使用多列布局
   - 不改变核心交互方式

5. **测试验证**
   - Chrome DevTools 移动端模拟
   - 真机测试
   - 检查触摸区域大小
   - 检查滚动性能

## 禁止行为

- 不要桌面端优先再适配移动端
- 不要使用小于 44px 的可点击区域
- 不要依赖 hover 交互
- 不要在移动端使用复杂表格
