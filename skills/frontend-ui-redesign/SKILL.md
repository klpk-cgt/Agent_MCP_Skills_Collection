# Frontend UI Redesign Skill

## 触发场景

- 资料页面重构
- 首页卡片优化
- 排行榜页面
- 竞猜页面
- AI 预测模块
- 移动端页面优化

## 执行流程

1. **阅读现有代码**
   - 阅读页面组件、子组件、样式文件
   - 理解数据结构（props、state、API 响应）
   - 确认业务逻辑和数据流

2. **确认不破坏的内容**
   - 已有接口和字段
   - 已有业务逻辑
   - 已有路由结构
   - 已有状态管理

3. **输出设计方案**
   - 视觉层级分析
   - 配色方案
   - 组件结构
   - 响应式断点
   - 交互动效

4. **修改组件和样式**
   - 优先移动端适配（375px / 390px）
   - 使用项目已有的 UI 库
   - 保持组件职责单一
   - 提取可复用组件

5. **保留原有业务逻辑**
   - 不修改 API 调用
   - 不修改状态管理
   - 不修改路由跳转
   - 不修改数据处理

6. **执行验证**
   - 运行 `build` / `lint` / `typecheck`
   - 检查控制台错误
   - 检查移动端样式

7. **总结输出**
   - 视觉变化说明
   - 修改文件清单
   - 新增组件说明
   - 潜在风险提示

## 禁止行为

- 不要随意改文案
- 不要随意改数据字段
- 不要大范围重构业务逻辑
- 不要引入新的状态管理方案
