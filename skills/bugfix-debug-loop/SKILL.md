# Bugfix Debug Loop Skill

## 触发场景

- 线上 Bug 修复
- 开发环境问题排查
- 测试反馈的 Bug
- 回归问题修复

## 执行流程

1. **复现问题**
   - 确认复现步骤
   - 记录环境信息（浏览器、OS、Node 版本）
   - 截图或录屏保存证据
   - 确认问题是否稳定复现

2. **定位报错来源**
   - 查看控制台错误信息
   - 查看网络请求状态
   - 查看服务端日志
   - 使用断点调试

3. **阅读相关代码**
   - 沿调用链追踪问题
   - 理解代码意图
   - 确认最近修改是否引入问题
   - 检查 `git log` / `git diff`

4. **提出最小修改方案**
   - 只修改导致问题的代码
   - 不做附带重构
   - 说明修改原因
   - 评估修改影响范围

5. **修改代码**
   - 小步修改
   - 保持代码风格一致
   - 添加必要的注释

6. **运行验证命令**
   ```bash
   npm run lint
   npm run build
   npm run test  # 如有
   ```
   - 手动验证问题是否已修复
   - 验证未引入新问题

7. **失败处理**
   - 如果验证失败，回到步骤 2
   - 根据新错误信息重新分析
   - 不要在同一方案上反复尝试

8. **输出报告**
   - 根因分析
   - 修改内容清单
   - 验证结果
   - 回归风险评估

## 禁止行为

- 不要凭空猜测原因
- 不要大范围重构
- 不要跳过验证步骤
- 不要隐藏失败结果
- 不要在未复现问题前修改代码
- 不要同时修改多个不相关的问题
