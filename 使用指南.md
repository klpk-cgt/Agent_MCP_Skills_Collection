# 📖 使用指南

## 什么是 Skill？

**Skill** 是 CodeBuddy 的领域扩展包，为 AI 编程助手提供专业知识、标准化工作流和可执行工具。每个 Skill 包含：

- **SKILL.md**：主配置文件，定义触发条件、工作流程和知识库
- **References**：领域参考文档，AI 按需读取
- **Scripts/Templates**：可执行脚本和代码模板

## 什么是 MCP？

**Model Context Protocol (MCP)** 是一种开放协议，让 AI 助手安全地与外部工具和服务交互。MCP Server 提供文件系统、数据库、浏览器、API 等能力。

## Skill 详解

### 1. Darwin Skill（达尔文.skill）

**用途**：自动优化 Skill 质量

**触发词**：`优化skill`、`skill评分`、`自动优化`、`达尔文`、`darwin`

**工作流程**：
1. 对 SKILL.md 进行 8 维度评分（满分 100）
2. 通过爬山算法自动迭代改进
3. Git 版本控制，只保留改进、自动回滚退步
4. 生成可视化成果卡片

**适用场景**：创建新 Skill 后需要优化、定期维护现有 Skill

### 2. GitHub Skill

**用途**：通过 `gh` CLI 管理 GitHub

**功能**：
- 管理 Issues 和 Pull Requests

- 搜索仓库和代码

### 3. Grill Me Skill

**用途**：深度追问式方案审查

**触发词**：`grill me`、`challenge my thinking`

**工作流程**：
1. 列出顶层决策分支（3-6 项）
2. 从最基础的开始逐层追问
3. 逐个解决决策树中的每个分支
4. 直到所有分支达成共识

**适用场景**：技术方案评审、架构设计讨论、计划压力测试

### 4. GSAP 动画开发助手

**用途**：生成和审查前端动画代码

**触发词**：`帮我做一个页面动效`、`用 GSAP 实现动画`、`滚动触发动画`

**支持框架**：React、Vue、Svelte、Vanilla JS

**参考文档（8 个）**：
- `gsap-core.md` — 基础 tween/easing/stagger
- `gsap-timeline.md` — 多步骤编排
- `gsap-scrolltrigger.md` — 滚动动画
- `gsap-plugins.md` — Flip/Draggable/SplitText 等
- `gsap-react.md` — React/Next.js 集成
- `gsap-performance.md` — 性能优化
- `gsap-frameworks.md` — Vue/Svelte 集成
- `gsap-utils.md` — 工具函数

---

### 5. Impeccable（前端设计工具集）

**用途**：创建高品质、非 AI 审美的前端界面

**触发词**：`界面设计`、`前端设计`、`响应式`、`动效`、`配色`、`设计评审` 等

**核心能力（20+ 场景）**：
- **视觉风格**：bolder（增强冲击力）、quieter（降噪）、colorize（策略性色彩）、overdrive（非凡效果）
- **布局排版**：arrange（修复布局）、typeset（改进字体）、adapt（响应式适配）
- **动效体验**：animate（添加动效）、delight（愉悦感）、onboard（引导设计）
- **质量保障**：audit（全面审查）、critique（UX 评审）、polish（打磨）、harden（加固）、optimize（性能优化）
- **设计系统**：extract（提取系统）、normalize（规范化）、distill（精简）、clarify（改进文案）

**核心原则**：
- 禁止 AI 审美（Inter 字体、纯黑 #000、AI 紫蓝色）
- 每次设计都不同，做出意想不到的创意选择
- 字体、色彩、动效都有专属的 DO/DON'T 指南

---

### 6. 前端开发

**用途**：全栈前端开发，结合 UI 设计、动效、AI 媒体生成

**五大能力**：
1. **Design Engineering** — 3 个可调旋钮（设计变异度/动效强度/视觉密度）
2. **Motion Engine** — 工具选择矩阵（Framer Motion/GSAP/Lottie/Three.js/CSS）
3. **Asset Generation** — MiniMax API 生成图片/视频/音频/TTS
4. **Copywriting** — 文案框架（AIDA/PAS/FAB）
5. **Visual Art** — p5.js 交互艺术/静态输出

**6 阶段工作流**：
1. 设计架构 → 2. 动效架构 → 3. 资源生成 → 4. 文案 → 5. 构建 UI → 6. 质量门

## MCP 配置指南

### 通用 MCP 配置结构

```json
{
  "mcpServers": {
    "<server-name>": {
      "command": "npx",
      "args": ["-y", "<package-name>"],
      "env": {
        "<ENV_VAR>": "<your-value>"
      },
      "type": "stdio",
      "disabled": false
    }
  }
}
```

### 各 MCP 模板说明

| 模板文件 | 包名 | 必需环境变量 |
|---------|------|------------|
| `github-mcp.json` | `@modelcontextprotocol/server-github` | `GITHUB_PERSONAL_ACCESS_TOKEN` |
| `filesystem-mcp.json` | `@modelcontextprotocol/server-filesystem` | 无（需指定允许的目录） |
| `fetch-mcp.json` | `mcp-fetch-server` | 无 |
| `puppeteer-mcp.json` | `@modelcontextprotocol/server-puppeteer` | 无 |
| `chrome-devtools-mcp.json` | `chrome-devtools-mcp` | 无 |
| `postgresql-mcp.json` | `@modelcontextprotocol/server-postgres` | `DATABASE_URL` |

### 安装配置步骤

1. **选择需要的 MCP Server**
2. **替换模板中的占位符**（Token、路径、数据库连接等）
3. **合并到你的 `mcp.json`**
4. **重启 CodeBuddy** 使配置生效

## 常见问题

**Q: 如何判断哪个 Skill 适合我的场景？**
A: 看触发词 — 每个 Skill 都在使用指南中列出了触发关键词。

**Q: 可以同时使用多个 Skill 吗？**
A: 可以，Skills 之间是独立的，可以同时加载多个。

**Q: MCP 配置不生效怎么办？**
A: 1) 检查 Token/凭据是否正确；2) 确认 `npx` 可以正常下载包；3) 重启 CodeBuddy
