# MCP 安全要求

1. API Key、Token、密码不能提交到 Git。
2. 所有敏感信息使用环境变量。
3. 数据库 MCP 默认只读。
4. 生产环境 MCP 不默认启用。
5. SSH、Shell、Docker、数据库写入权限必须人工确认。
6. 不允许 Agent 自动删除数据库、volume、生产文件。
7. 不允许 Agent 自动推送到主分支。
8. 不允许 Agent 在不了解项目结构时直接大改。

## 高权限 MCP 清单

以下 MCP 需要特别注意，建议仅在具体项目按需启用：

| MCP | 风险等级 | 说明 |
|-----|---------|------|
| SSH | 🔴 高 | 可远程执行服务器命令 |
| Shell | 🔴 高 | 可执行任意 shell 命令 |
| Docker | 🟡 中 | 可操作容器和镜像 |
| PostgreSQL | 🟡 中 | 可读写数据库 |
| Supabase | 🟡 中 | 可读写数据库 |
| SQLite | 🟡 中 | 可读写本地数据库 |
| Vercel | 🟡 中 | 可管理部署和域名 |
| Sentry | 🟢 低 | 主要为只读查询 |
